### Git 提交规范

#### 提交结构

```sh
<type>(<scope>): <subject>
<body>
<footer>
```

```sh
git commit -m "feat(call): 支持群呼功能"
```

##### 提交类型 type

```sh
feat      # 新功能
fix       # bug 修复
docs      # 文档更新
style     # 格式调整
refactor  # 重构（非 fix、非 feat）
perf      # 性能优化
test      # 测试文件更新
chroe     # 构建/依赖更新
ci        # CI 配置更新
revert    # 回滚
```

##### 模块 scope

表示当前提交影响的模块

##### 标题 subject

要求：

- 简单明了
- 祈使句
- 无需句号结尾

##### 主体 body

用于说明动机、实现方式、额外上下文信息

##### 脚注 footer

关联 issue

```sh
git commit -m "fix(call): 修复并发呼叫账户无法注册bug (#11)"
```

#### 原子提交

一次提交只做一件事

错误示例（一次提交包含多个逻辑）：

```sh
feat: add login & fix logout & update docs
```

正确示例：

```sh
feat(login): add login API
fix(logout): handle null token
docs(auth): update README
```
