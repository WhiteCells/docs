冯•诺依曼结构计算机的基本思想是什么？

按此思想设计的计算机硬件系统应由哪些部件组成？

**基本思想**：顺序存储概念、顺序执行、二进制编码、输入/输出设备、控制单元

**主要部件**：存储器、运算器、控制器、输入设备、输出设备

**为什么计算机中采用二进制进行数据表示和运算？**

1）二进制可以表示任何数据信息；2）二进制数码最少，方便对应简单的物理状态；3）运算规则简单，易实现算术逻辑运算电路

---

总线传输中**并行传输**和**串行传输**的优缺点

**串行传输**：**优点**：数据同时通过多个导线传输，可以提高数据传输速率；**缺点**：随着数据宽度增加，所需要的导线数量和复杂性也增加，可能导致信号同步问题

**串行传输**：**优点**：数据逐位通过单一导线传输，减少了所需的导线数量。简单、成本低，适用于长距离通信；**缺点**：传输速度较慢，因为数据逐位传输

设主存容量为 16 MB，按字节寻址；虚拟存储器容量为 4 GB，采用页式虚拟存储器，页面大小为 4 KB。完成下列各问：

**(1)计算物理页号、页内偏移、虚拟页号字段各为多少位。**

物理页号位数：主存容量： $16MB=2^{24}Byte$，页面大小： $4KB=2^{12}Byte$，物理页号需要： $24-12=12Bit$

页内偏移位数：页面大小： $4KB=2^{12}Byte$，页内偏移需要： $12Bit$

虚拟页号位数：虚拟存储器容量： $4GB=2^32Byte$，页面大小： $4KB=2^{12}Byte$，虚拟页号需要：$32-12=20Bit$

**(2)计算页表中页表项的数量。**

虚拟存储器容量： $4GB=2^{32}Byte$，页面大小： $4KB=2^{12}Byte$，虚拟页表需要： $2^{32}/2^{12}=2^{20}$ 个页表项

| 虚页号 | 有效位 | 实页号 | 虚页号 |

| 00010H | 0      | 002H   | 03FFFH |
| 00015H | 1      | 035H   | ...    |

虚拟地址 (00015240)H：

物理地址： $035H \times 4KB + 240H = (035 \times 1024 + 240)H= (036240)H$

虚拟地址 (03FFF180)H：

虚拟页号为 03FFFH，对应有效位为 1，所以该页面存在于主存中。

$双地址指令剩余状态数：2^4-15=1$；$单地址指令剩余状态数：1×2^6-34=30$

$零地址指令的数目最多为：30×2^6=1920条$

假设一台计算机主频为 1 GHZ，在其上运行由 $2\times10^5$ 条指令组成的目标代码，程序主要由4类指令组成，他们所占的比例和各自的 CPI 如下表所示，求程序的CPI和MIPS。

| 指令类型      | CPI  | 指令比例 |
| ------------- | ---- | -------- |
| 算术和逻辑    | 1    | 60%      |
| Load/Store    | 2    | 18%      |
| 转移          | 4    | 12%      |
| Cache缺失访存 | 8    | 10%      |

**求程序的CPI**

$CPI=1\times 0.6 + 2\times 0.18 + 4 \times 0.12 + 8\times 0.1=2.24$

**求程序的MIPS。**

$MIPS=\frac{1000}{2.24}\approx 446.43$

**求程序执行时间？**

$执行时间=\frac{2\times 10^5\times 2.24}{10^9}=0.000448s$

**CPU全性能公式最早由Intel提出，请用CPU全性能公式解释计算机性能与指令的CPI和主频的关系。**

$CPU时间=\frac{指令数\times CPI}{主频}$

主频越高，CPI 越低，计算机性能越强

字长16位，主存64K，指令单字长单地址，80条指令。寻址方式有直接、间接、相对、变址。请设计指令格式。

1. 分配位给寻址方式

   有四种寻址方式，所以需要至少 2 位来表示寻址方式（$2^2=4$）

2. 分配位给指令操作码

   有 80 条指令，所以需要需要至少 7 位来表示 80 条指令（$2^7 = 128 > 80$）

3. 分配位给地址

   在16位的指令中，已经使用了2位（寻址方式）和7位（操作码），所以剩下 7 位用于地址字段

4. 指令格式

   `[ 7位操作码 ] [ 2位寻址方式 ] [ 7位地址 ]`

字长假定在一个 8 位字长的计算机中运行如下C程序段：

```c
unsigned int x=134;
unsigned int y=246;
int m=x;
int n=y;
unsigned int z1 = x-y;
unsigned int z2 = x+y;
int k1=m-n;
int k2=m+n;
```

若编译器编译时将8个8位寄存器R1~R8分别分配给变量x、y、m、n、z1、z2、k1和k2.请回答下列问题（提示：有符号整数用补码表示）。

**（1）执行上述程序段后，寄存器R1、R5和R6的内容分别是什么（用十六进制表示）？**

x = 134，y = 246，分别在8位无符号整数中表示为 0x86 和 0xF6

将x和y存入寄存器：R1：x = 134 (0x86)，R2：y = 246 (0xF6)

z1 = x - y：134 - 246 = -112（对于8位无符号整数，结果是256 - 112 = 144，表示为0x90）

z2 = x + y：134 + 246 = 380（对于8位无符号整数，结果是380 - 256 = 124，表示为0x7C）

因此：R5：z1 = 144 (0x90)，R6：z2 = 124 (0x7C)

**（2）执行上述程序段后，变量m和k1的值分别是多少（用十进制表示）？**

变量 m 和 n 是有符号整数：

m = x：x = 134，表示为有符号整数，即 -122（0x86的有符号表示为-122）。

n = y：y = 246，表示为有符号整数，即 -10（0xF6的有符号表示为-10）。

计算有符号整数减法：

k1 = m - n：-122 - (-10) = -122 + 10 = -112。

因此：m 的值是 -122（十进制），k1的值是 -112（十进制）

**（3）上述程序段涉及有符号整数加减、无符号整数加减运算，这四种运算能否利用同一个加法器辅助电路实现？简述理由。**

可以。加法器只进行二进制的加法运算，不关心操作数是否有符号；加法器的结果可以通过解释来得到无符号和有符号的结果；减法运算可以通过加法器实现，通过将减数取补码再进行加法运算。

**（4）计算机内部如何判断有符号整数加减运算的结果是否发生溢出？上述程序段中，哪些有符号整数运算语句的执行结果会发生溢出？**

加法时，如果两个正数相加得到负数，或者两个负数相加得到正数，则发生溢出。

减法时，如果一个正数减去负数得到负数，或者一个负数减去正数得到正数，则发生溢出。

在上述程序段中：

m = -122 和 n = -10。

k1 = m - n：-122 - (-10) = -112，没有发生溢出。

k2 = m + n：-122 + (-10) = -132，会发生溢出，因为 8 位有符号整数范围是 -128 到 127。

因此，k2的计算会发生溢出。
