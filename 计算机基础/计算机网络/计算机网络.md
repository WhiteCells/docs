### 1.1 网络、互联网与因特网

- 网络（Network）由若干**结点**（Node）和连接这些结点的**链路**（Link）组成。
- 多个网络可以通过**路由器**连起来，即互联网（或互连网）。
- 因特网（Internet）是世界上最大的互联网络。

> internet，为通用名词，互连网（互联网），任意通信协议。
>
> Internet，为专有名词，因特网，必须 **TCP/IP** 协议作为通信规则。

因特网的三个阶段

1. 1969 ARPANET 向互联网发展
   1. 70 年代中期，研究多种网络互连。
   2. **1983年**，**TCP/IP** 协议成为 ARPANET 的标准协议（**因特网诞生时间**）。
2. 1985 三级结构因特网
   1. NSFNET（主干网、地区网和校园网）。
   2. 1990年，ARPNET 完成任务，正式关闭。
   3. 1991年，因特网初步商业化，开始收费。
3. 1993 多层次 ISP 结构的因特网
   1. 1993年，NSFNET被替代，由各种 ISP 运营。
   2. 1994年，**www技术**推动因特网迅速发展。
   3. 1995年，NSFNET停止运作，因特网彻底商业化。

因特网服务提供商（Internet Service Provider，ISP）

用户通过 ISP（Mobile、Unicom、Telecom等） 接入因特网

ISP 可以从因特网管理机构申请到成块的 **IP** 地址，同时拥有通信线路以及路由器等连网设备，因特网上的主机必须有 **IP** 地址才能进行通信。

因特网已发展成为基于 ISP 的多层次结构的互联网络。

某个用户接入互连网，那他也可以成为一个 ISP。

**因特网的组成**

- 边缘部分

  由所有连接在因特网上的**主机**组成，这部分是用户直接使用的，用来进行**通信**和**资源共享**。

- 核心部分

  由**大量网络**和连接这些网络的**路由器**组成，这部分是为边缘部分提供服务的（提供连通性和交换）

### 1.2 三种交换方式

1. 电路交换（Circuit Switching）
2. 分组交换（Packet Switching）
3. 报文交换（Message Switching）

#### 1.2.1 电路交换

> n 部电话机两两相连，需要多少电线？ $\frac{n(n-1)}{2}$
>

使用中间设备（电话交换机）就只需要 n 个电线。

- 电话交换机接通电话线的方式称为电路交换。
- 交换是按照某种方式动态地分配传输线路的资源。
- 电路交换的三个步骤：
  1. 建立连接（分配通信资源）
  2. 通话（占用通信资源）
  3. 释放连接（归还通信资源）

优点：

1. 通信时延小
2. 有序传输
3. 没有冲突
4. 适用范围广（数字信号、模拟信号）
5. 实时性强
6. 控制简单

缺点：

1. 建立连接时间长
2. 线路独占，使用效率低
3. 灵活性差
4. 难以规格化（类型、规格、速率的设备不同）

#### 1.2.2 分组交换

在因特网中分组交换机就是路由器。



主机1 向另 主机2 发送信息，该消息的整块数据被称为一个报文，较长的报文划分成一个个更小的等长数据段。每一个数据段前面加上由**必要的控制信息**组成的首部后构成一个分组（包）。

> 发送方 主机1，需要进行构建分组和发送分组。

分组交换机将分组暂存，检查其首部中的目的地址进行查表转发，找到合适的转发接口，通过接口转发给下一个分组交换机。

> 分组交换机 路由器，需要缓存分组和转发分组。

主机2 接受到分组后去掉首部，将各数据段组合还原出原始报文。

> 接收方 主机2，接受分组，还原报文。

优点

1. 无需建立连接
2. 线路利用率高
3. 简化存储管理（分组长度固定，缓冲区大小固定）
4. 加速传输（后一个分组的存储操作和前一个分组的转发操作可以同时进行）
5. 减少出错率和重发数据量

缺点

1. 引起转发时延
2. 需要传输额外的信息量
3. 对于数据报服务，存在时丢失或重复分组的问题，对于虚电路服务，存在呼叫建立、数据传输和虚电路释放三个过程

#### 1.2.3 报文交换

报文交换中的交换结点也采用存储转发方式，报文交换对报文大小没有限制，需要交换机有较大的空间。（主要用于早期的电报通信网）

优点

1. 无需建立连接
2. 动态分配线路
3. 提高线路可靠性
4. 提高线路利用率
5. 提供多目标服务

缺点

1. 转发时延（结点交换机上经历存储转发的过程）
2. 需要较大的存储缓存空间
3. 需要传输额外的信息量



### 1.3 计算机网络的分类

#### 1.3.1 按交换技术分类

- 电路交换网络
- 报文交换网络
- 分组交换网络

#### 1.3.2按使用者分类

- 公用网（所有按电信公司的规定交纳费用的人都可以使用的网络）
- 专用网（某个部门为某单位（军队、铁路等）的特殊业务工作的需要而建造的网络，不向本单位以外的人提供服务）

#### 1.3.3 按传输介质分类

- 有线网络（双绞线网络、光纤网络等）
- 无线网络

#### 1.3.4 按覆盖范围分类

- 广域网 WAN（几十公里~几千公里）（因特网的核心）
- 城域网 MAN（5~50 公里）城市主干网
- 局域网 LAN（1 公里左右）
- 个域网 PAN（10 米左右）（便携式计算机、打印机、鼠标、键盘等用无线技术连接起来的网络）

#### 1.3.5 按拓扑结构分类

- 总线型网络（单根传输线，把计算机连接起来，建网容易，增减结点方便，节省线路，重负载时效率低，仍以一处出故障，全网所有设备瘫痪）
- 星型网络（每个计算机与中央设备相连，中央设备早期是计算机，后来是集线器，现在一般是交换机或路由器，便于网络的集中控制和管理，成本高，中央设备对故障铭感）
- 环型网络（将所有计算机的网络接口连接成一个环，环中信号单向传输）
- 网状型网络（每个结点至少由两个路径与其他结点相连，可靠性高，成本高，控制复杂）



### 1.4 计算机网络性能指标

- 速率
- 带宽
- 吞吐量
- 时延
- 时延带宽积
- 往返时间
- 利用率
- 丢包率

#### 1.4.1 速率

##### 1.4.1.1 数据量单位

比特，单位 bit，简写为 b，1 bit 是二进制的一个 0 或 1。

字节，单位 Byte，简写 B。

8 bit = 1 Byte（8 比特位 = 1 字节）

$KB = 2^{10} B$

$MB=K·KB=2^{10}·2^{10}B=2^{20}B$

$GB=K·MB=2^{10}·2^{20}B=2^{30}B$

$TB=K·GB=2^{10}·2^{30}B=2^{40}B$

> 硬盘等存储设备中的 $GB$ 是 $10^{9}B$，操作系统进制单位是 $2^{30}B$

熟虑是连接在计算机网络上的主机在数字信道上的传送比特的速率，也称**比特率**或**数据率**。

##### 1.4.1.2 数据率单位

$bit/s(b/s、bps)$

$kb/s=10^{3}b/s(bps)$

$Mb/s=k·kb/s=10^{3}·10^{3}b/s=10^{6}b/s(bps)$

$Gb/s=k·Mb/s=10^{3}·10^{6}b/s=10^{9}b/s(bps)$

$Tb/s=k·Gb/s=10^{3}·10^{9}b/s=10^{12}b/s(bps)$

> 发送大小为 $100MB$ 数据块，网卡的发送率为 $100Mbps$，需要多长时间？
>
> $\frac{100(MB)}{100(Mb/s)}=\frac{MB}{Mb/s}$
>
> 严格来说 $M$ 是不能约掉的，分子上的 $M$ 是数据量单位（$2^{20}$），分母上的 $M$ 是数据率单位（$10^6$）
>
> $\frac{2^{20}(B)}{10^6(b/s)}=\frac{2^{20}\times8(b)}{10^6(b/s)}=\frac{2^{23}}{10^6}(s)=8.388608(s)$
>

#### 1.4.2 带宽

##### 1.4.2.1 在模拟信号系统中的意义

信号所包含的各种不同频率成分所占据的**频率范围**。

单位 Hz（kHz、MHz、GHz）

##### 1.4.2.2 在计算机网络中的意义

网络的**通信线路**所能传输数据的能力，因此网络带宽表示在单位时间内从网络中的某个点到另一点所能通过的最**数据量**。

单位 b/s（kb/s、Mb/s、Gb/s、Tb/s）

> 常说的带宽为 几百兆，这里省略了比特每秒。

> 一条通信线路的频率宽度越宽，能传输数据的最高数据率也越高。

#### 1.4.3 吞吐量

单位时间内，通过每个网络（或信道、接口）的数据量。

吞吐量被经常用于对现实世界终端网络的一种测量，以便知道实际上到底有多少数据量能通过网络。

吞吐量受网络的**带宽**和**额定速率**的限制。

> 带宽是理想，吞吐量是现实。

#### 1.4.4 时延

分组机传送到目的主机的过程中，产生时延的部分

1. 源主机将分组发往传输线路（**发送时延**）
2. 分组的电信号在链路上传输（**传输时延**）
3. 路由器收到分组后对其存储和转发（**处理时延**）

> 源主机和目的主机之间的路径会由多段链路和多个路由构成，因此会有多个**传输时延**和**处理时延**。

> 遵循**木桶效应**，在网卡、信道带宽、接口，最终的速率就是其中短板的速率。
>
> 在构建网络时应尽可能各设备间以及传播介质的传输速率匹配。

**发送时延**的计算公式 $\frac{分组长度(b)}{发送速率(b/s)}$

**传播时延**的计算公式 $\frac{信道长度(m)}{电磁波传播速率(m/s)}$

电磁波在自由空间的传播速率是光速 $3\times10^8m/s$

在无线电缆中的传播速率约为 $2.3\times10^8m/s$

在光纤中的传播速率约为 $2.0\times10^8m/s$

数据块长度 $100MB$

信道带宽 $1Mb/s$

传送距离为  $1000km$

计算发送时延和传播时延

发送时延 =  $\frac{100\times2^{20}\times8(b)}{10^6(b/s)}=838.8608(s)$

传播时延 =  $\frac{1000\times10^3(m)}{2.0\times10^8(m/s)}=0.005(s)$

此时**发送时延**站主导

数据块长度 $1B$

信道带宽 $1Mb/s$

传送距离为 $1000km$

计算发送时延和传播时延

发送时延 = $\frac{1\times8(b)}{1\times10^6(b/s)}=8\times10^{-6}(s)$

传播时延 = $\frac{1000\times10^3(m)}{2.0\times10^8(m/s)}=0.005(s)$

此时**传播时延**站主导

#### 1.4.5 时延带宽积

这里的时延是**传播时延**

时延带宽积 = 传播时延 * 带宽

> 时延带宽积看成管道，传播时延比作管道长，带宽比作管道面积，时延带宽积就是管道体积。

- 若发送端连续发送数据，则在做发送的第一个比特即将发送到达终点时，发送端就已经发送了时延带宽积个比特。
- 链路的时延带宽积又称为**以比特为单位的链路长度**

#### 1.4.6 往返时间

往返时间 RTT（Round-Trip Time）

> 从源主机发送分组开始，直到源主机接受到来自目的主机的确认分组为止，所需要的时间就是往返时间。

#### 1.4.7 利用率

1. 信道利用率（表示某信道有百分之几的时间是被利用的（被数据通过））
2. 网络利用率（全网的信道利用率的加权平均）

> 根据排队论，当某信道的利用率增大时，该信道引起的的时延也会迅速增加。因此信道利用率并非越高越好
>
> 如果用  $D_{0}$  表示网络空闲时的时延， $D$ 表示网络当前的时延，那么在适当的假定条件下，可以用下面的简单公式来表示 $D$、 $D_0$ 和利用率 $U$ 之间的关系: 
>
>  $D=\frac{D_0}{I-U}$ 
>
> 当网络的利用率达到 50% 时，时延加倍；
>
> 当网络的利用率超过 50% 时，时延急剧增大；
>
> 当网路的利用率接近 100% 时，时延就趋于无穷大；
>
> 因此，拥有较大主干网的 ISP 通常会控制他们的信道利用率不超过 50%，如果超过了，就要准备扩容，增大线路的带宽。
>
> 也不能使信道利用率太低，这会使信道资源浪费，应该使用一些机制，可以根据情况的动态调整输入到网络中的通信量，使网络利用率保持在一个合理的范围。

#### 1.4.8 丢包率

即分组丢失率，是指在一定的时间范围内，传输过程中**丢失的分组数量与总分组数量的比率**。

丢包率可分为接口丢包率、结点丢包率、链路丢包率、路径丢包率、网络丢包率等。

丢包率是网络运维人员十分关心的性能指标，对于普通用户往往不关心，因为意识不到。

**分组丢失主要有两种情况：**

1. 分组在传输的过程中出现**误码**，被结点丢失。
2. 分组到达一台队列已满的分组交换机时被丢弃，在通信量较大时就可能造成**网络拥塞**。（实际上路由器根据据自身拥塞控制方法，在输入缓存还未满的时候就丢失分组）

**丢包率反映了网络的拥塞程度：**

1. 无拥塞时路径丢包率为 0
2. 轻度拥塞时路径丢包率为 1%~4%
3. 严重拥塞时路径丢包率为 5%~15%

### 1.5 计算机网络体系结构

#### 1.5.1 常见的计算机网络体系结构

##### 1.5.1.1 OSI 体系结构

法律上的国际标准

开放系统互连参考模型，简称 **OSI 体系结构**

是一个 7 曾协议的体系结构（从下向上）

1. 物理层
2. 数据链路层
3. 网络层
4. 运输层
5. 会话层
6. 表示层
7. 应用层

> OSI 标准失败的原因：
>
> 1. OSI 专家们缺乏实际经验，完成 OSI 标准时没有商业驱动力。
> 2. OSI 协议实现起来过分复杂，而且运行效率低。
> 3. OSI 指定周期长，使得按 OSI 标准生产的设备无法及时进入市场。
> 4. OSI 的层次划分也不太合理，有些功能在多个层次中重复出现。

##### 1.5.1.2 TCP/IP 体系结构

事实上的国际标准

因特网从 1983 年开始使用 **TCP/IP** 协议族，并逐步演变成 TCP/IP 参考模型，该模型是一个 4 层协议的体系结构（从下向上）

1. 网络接口层
2. 网际层
3. 运输层
4. 应用层

> TCP/IP 体系结构相当于将 OSI 体系结构的**物理层**和**数据链路层**合并为**网络接口层**，去掉的了**会话层**和**表示层**。
>
> 由于 TCP/IP 在**网络层**使用的协议是 IP 协议（网际协议），因此 TCP/IP 体系结构的网路层常称为网际层。

> 接入因特网的主机必须使用 TCP/IP 协议，即不需要接入因特网也可以使用 TCP/IP 协议。
>
> 在用户操作的主机中通常都带有 TCP/IP 体系结构标准的 TCP/IP 协议族。用于网络互联的路由器中，也带有符合 TCP/IP 体系结构标准的 TCP/IP 协议族，但其一般只包含**网络接口层**和**网际层**。TCP/IP 体系结构的网络接口层并没有规定具体内容，可以互联全世界不同的网络接口（有线以太网接口、无线局域网WiFi接口），本质上 TCP/IP 体系结构只有上面的三层。
>
> - IP 协议是 TCP/IP 体系结构**网际层**的核心协议，负责互联不同的网络接口，为网络应用提供服务；
>
> - TCP 和 UDP 是 TCP/IP 体系结构**运输层**的两个重要协议；
> - TCP/IP 体系结构应用层包含的大量的应用层协议（HTTP、SMTP、DNS、RTP等）

##### 1.5.1.3 原理体系结构

结合 OSI 和 TCP/IP 的优点，采用一种 5 层协议的原理体系结构（从下到上）

1. 物理层
2. 数据链路层
3. 网路层
4. 运输层
5. 应用层

> 原理体系结构将 TCP/IP 协议中的**网络接口层**重新划分成**物理层**和**数据链路层**。

#### 1.5.2 计算机网络体系结构分层的必要性

> 计算机网络是一个非常复杂的系统，早在最初的 ARPANET 设计时就提出了分层的设计理念。
>
> “分层”可将庞大而复杂的问题，转化为若干较小的局部问题，而这些较小的局部问题就易于研究和处理。

**物理层**

考虑两台电脑连接所需要解决的问题

- 采用什么传输媒体
- 采用什么物理接口
- 采用什么信号表示比特 0 和 1

**数据链路层**

考虑在一个总线型网络中，主机A 向主机B 发送信号，主机B 怎么知道这个信号是发送给自己的去接受，同时其他主机直到不是发给自己去拒接。

- 如何标识网络中的各主机（主机编址问题，例如 MAC 地址）
- 如何从信号所表达的一连串比特流中区分地址和数据（分组的封装格式问题）
- 如何协调各主机争用总线

**网络层**

考虑在一个有多个路由器的多个网络中

- 如何标识个网络以及网络中的各主机（网络和主机共同编织问题，例如 IP 地址）
- 路由器如何转发分组，如何进行路由选择的问题

**运输层**

主机收到来自服务器的分组，这些分组应该交给那个应用进程处理

- 如何解决进程之间基于网络的通信问题

#### 1.5.3 计算机网络体系结构分层思想举例

主机和 Web 服务器之间基于网络的通信，实际上是主机中的浏览器应用进程与 Web 服务器应用进程之间基于网络的通信。



1. 浏览器进程

应用层按 HTTP 协议规定，构建一个 HTTP 请求报文。

应用层将 HTTP 请求报文交付给运输层处理。

运输层给 HTPP 请求报文添加 TCP 首部，使之称为 TCP 报文段，该首部的作用主要是为了区分应用进程以及实现可靠传输。

运输层将 TCP 报文段交付给网络层处理。

网络层给 TCP 报文段添加一个的 IP 首部，使之称为 IP 数据报，该首部的作用主要是为了使 IP 数据报可以在互连网上传输，也就是被路由器转发。

网络层将 IP 数据报交付给数据链路层处理。

数据链路层给 IP 数据报添加一个首部和一个尾部，使之称为帧，该首部的作用主要是为了让帧能够在一段链路上或一个网络上传输，能够被相应的目的主机接受，该尾部的作用是为了让目的主机检查所接受到的帧是否有误码。

数据链路层将帧交付给物理层。

物理层将帧看作比特流。由于网络 N1 是以太网，因此物理层会给比特层前面加上前导码，为了让目的主机做好接受帧的准备。

物理层将添加有前导码的比特流变换成相应的信号发送到传输媒体。

信号通过传输媒体到达路由器。
